# 오늘 앱 (Onul App) - 프로젝트 개요

## 1. 프로젝트 소개

청소 서비스 클라이언트를 위한 프로젝트 관리 플랫폼

### 핵심 가치
- 대형/소형 프로젝트의 체계적 관리
- 참가자(마스터) 효율적 배치 및 관리
- 실시간 소통을 통한 원활한 협업
- 비포/애프터 사진으로 품질 증명
- 포트폴리오 자동 축적

---

## 2. 프로젝트 구조

```
대형 프로젝트 (예: "A 쇼핑몰 전체 청소")
    │
    ├── 소형 프로젝트 1 (예: "1층 매장 구역")
    │       ├── 참가자 A (마스터)
    │       ├── 참가자 B (마스터)
    │       └── 참가자 C (마스터)
    │
    ├── 소형 프로젝트 2 (예: "2층 푸드코트")
    │       ├── 참가자 D (마스터)
    │       └── 참가자 E (마스터)
    │
    └── 소형 프로젝트 3 (예: "주차장")
            └── 참가자 F (마스터)
```

---

## 3. 사용자 역할

### 3.1 최고 관리자 (Super Admin)
- 전체 시스템 관리
- 마스터 추가/수정/삭제
- 마스터별 역량 및 프로젝트 이력 확인
- 완료된 프로젝트 채팅 기록 열람
- 포트폴리오 관리

### 3.2 프로젝트 책임자 (Project Manager)
- 대형/소형 프로젝트 생성 및 관리
- 참가자 신청 승인/거절
- 프로젝트 진행 상황 모니터링
- 작업 완료 사진 검토 및 최종 승인

### 3.3 마스터 (Master/참가자)
- 프로젝트 신청
- 작업 수행
- 비포/애프터 사진 업로드
- 특이사항 기록
- 프로젝트별 채팅 참여

### 3.4 클라이언트 (Client/사용자)
- 청소 서비스 신청
- 진행 상황 확인
- 비포/애프터 사진 확인
- 완료 확인

|------|------|

| 언어 | TypeScript |

| 스토리지 | Supabase Storage |

| UI | React Native Paper 또는 NativeWind |

## 5. 프로젝트 워크플로우

```
1. 클라이언트 서비스 신청
        ↓
2. 최고관리자/책임자가 대형 프로젝트 생성
        ↓
3. 소형 프로젝트로 분할 (작업 구역별)
        ↓
4. 마스터들이 소형 프로젝트에 참가 신청
        ↓
5. 책임자가 참가 승인
        ↓
6. 프로젝트 시작 → 실시간 채팅 활성화
        ↓
7. 마스터들 작업 수행
        ↓
8. 작업 완료 시 비포/애프터 사진 + 특이사항 업로드
        ↓
9. 책임자 검토 후 최종 승인
        ↓
10. 클라이언트 완료 확인 → 프로젝트 종료
        ↓
11. 포트폴리오에 자동 축적
```

## 6. 주요 화면 구성

### 마스터용 앱
| 탭 | 기능 |
|----|------|
| 대시보드 | 진행 중 프로젝트, 완료 프로젝트, 프로필 관리 |
| 프로젝트 | 신청 가능 프로젝트 목록, 신청하기 |
| 매장관리 | 계약 매장별 정기 관리 |
| 채팅 | 프로젝트별 실시간 채팅 |

### 최고 관리자용
| 메뉴 | 기능 |
|------|------|
| 대시보드 | 전체 현황, 통계 |
| 프로젝트 관리 | 대형/소형 프로젝트 생성/관리 |
| 마스터 관리 | 마스터 추가/수정/삭제, 역량/이력 확인 |
| 포트폴리오 | 완료 프로젝트 모음, 외부 공유 |

### 클라이언트용
| 화면 | 기능 |
|------|------|
| 내 프로젝트 | 신청한 서비스 진행 상황 |
| 비포/애프터 | 작업 전후 사진 비교 |
| 완료 확인 | 서비스 완료 승인 |
